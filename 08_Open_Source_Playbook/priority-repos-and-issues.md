# Priority Open Source Targets (Starter to Advanced)

## How to Use

1. Pick one repository from Tier 1.
2. Start with docs/tests/small fixes.
3. Move to Tier 2 when 5+ PRs are merged.
4. Track every PR in contribution-tracker.md.

## Tier 1 (High Merge Probability)

- freeCodeCamp/freeCodeCamp
  - Contribution types: curriculum docs, test improvements, typo fixes, examples.
- firstcontributions/first-contributions
  - Contribution types: docs quality and onboarding clarity.
- public-apis/public-apis
  - Contribution types: API entry corrections and categorization updates.
- mdn/content
  - Contribution types: web docs improvements and example fixes.

## Tier 2 (Engineering Depth)

- expressjs/expressjs.com
  - Contribution types: docs, examples, migration notes.
- axios/axios
  - Contribution types: docs + focused bugfixes + tests.
- chartjs/Chart.js
  - Contribution types: docs, examples, issue reproduction.
- tanstack/query
  - Contribution types: docs and typed examples.

## Tier 3 (Advanced Visibility)

- vercel/next.js
- microsoft/TypeScript
- nodejs/node

Start Tier 3 only after strong consistency in Tier 1 and Tier 2.

## Weekly Contribution Target

- 3 small PRs (docs/tests/fixes)
- 1 medium PR (bugfix/refactor)
- 2 quality issues with reproducible reports

## PR Quality Standard

- Clear problem statement
- Minimal and clean diff
- Tests or verification evidence
- Respect repository contribution guidelines
